Now that we have our line of code in a tokenised form, we need to understand what the code is asking us to do.

For this, we'll need a parser. For this post, I'll be making a parser that only parses 'expressions.' Because SIMPLE's syntax is not very complicated compared to something like a C langauge, parsing it's statements, like PRINT, is somewhat arbitrary and will be described in the third post about the interpreter.

An expression is any complex equation that can be resolved to a singular value. For example 1 + 2 is an expression which resolves to 3, and 2 is an expression that resolves to 2. With these examples, finding the solution is simple enough, but for a more complex expression such as 1 + 3 * 2 - (3 + 5), we need to know what order we should be performing operations.

For a computer to know what to do, we use what is called an 'abstract syntax tree'. For the example 1 + 3 * 2 - (3 + 5), it would look something like this: 

[-]
 |-----[+]
 |      |-----(1)
 |      |-----[*]
 |             |-----(3)
 |             |-----(2)
 |-----[+]
        |-----(3)
        |-----(5)

In this tree, operations are performed from right to the left. It starts by multiplying 3 and 2, then adding 1 to that result, and so on up the tree.

To create this tree, we'll be using an algorith called Pratt Parsing. It is an 'operator precedence' parser. You can watch an excellent visual explanation of how the algorithm works in this video by Core Dumped.
